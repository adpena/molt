# Fast Math Generated Result Types

## Summary

Moved fast-math type seeding off a private ConstFloat/ConstInt/ConstBool opcode
match and onto the generated operand-independent result-type helper.

## Structural Changes

- Updated `fast_math.rs` to consume
  `opcode_operand_independent_result_tir_type`.
- Kept fast-math responsible for function-level opt-in, F64 eligibility, and op
  annotation policy.
- Extended `tests/test_gen_op_kinds.py` so fast-math is covered by the existing
  operand-independent result-type generated-fact contract.
- Updated design/spec routing docs to list fast-math as a consumer of generated
  result-type facts.
- Regenerated the structural audit board.
