# Range Devirt Generated Roles

## Summary

Moved `range_devirt.rs` pattern-scanner opcode membership to a generated
op-kind role table.

## Structural Changes

- Added `range_devirt_roles` to `runtime/molt-tir/src/tir/op_kinds.toml`.
- Added a dedicated generator validator that rejects unknown fields,
  duplicate/unknown opcodes, `none` roles, and role/opcode drift.
- Generated `RangeDevirtRole` and `opcode_range_devirt_role_table`.
- Updated `range_devirt.rs` to consume generated roles for
  CallBuiltin/GetIter/IterNextUnboxed pattern scanning.
- Kept live semantic checks in `range_devirt.rs`: builtin name, range arity,
  operand/result shape, loop-header role, dominance/setup placement, and CFG
  validation.
- Added generated-fact tests for table rows, generated Rust arms, production
  pass delegation, and validator drift rejection.
- Updated op-kind registry/spec docs and regenerated the structural audit board.
