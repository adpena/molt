# DFlash Research Refresh - 2026-06-24

## Current Source-Of-Truth

- DFlash is block-diffusion speculative decoding, not generic speculative
  decoding. The drafter predicts a full block in one forward pass, while the
  target autoregressive model verifies the block in parallel and accepts the
  longest valid prefix.
- The core fidelity requirements are: trained target-specific drafter,
  drafter/verifier separation, target hidden-state conditioning, KV injection
  into draft layers, non-causal in-block draft attention, block verification,
  and lossless target distribution preservation.
- The official z-lab repo and model cards now expose broad pretrained DFlash
  draft models and backend recipes for SGLang, vLLM, Transformers, and MLX.
  A DFlash draft checkpoint is not a standalone model; it must be paired with
  the matching target model.

## Latest Updates

- DFlash paper: arXiv:2602.06036, first posted 2026-02-05. It reports over
  6x lossless acceleration and up to 2.5x higher speedup than EAGLE-3 by
  replacing autoregressive drafting with a lightweight block diffusion drafter
  conditioned on target model context features.
- SGLang/LMSYS implementation writeup, 2026-06-15: DFlash injects target
  features into the draft model KV cache so deeper draft layers stay strongly
  conditioned. SGLang added a DFlashWorker/DFlashDraftModel path and Spec V2
  scheduling/overlap work around it.
- vLLM Speculators docs: DFlash uses a small diffusion draft model, Qwen3-style
  draft layers, a non-causal attention mask over verifier hidden states and mask
  embeddings, anchor/block prediction, and target verification of the longest
  valid prefix. vLLM docs mark it active development.
- z-lab/dflash repo: MIT, roughly 5k+ stars, 90 commits at the time of review,
  no releases, many supported draft checkpoints, and a stated plan to
  open-source training recipes.
- DDTree, arXiv:2604.12989, extends DFlash by building a draft tree from
  per-position block-diffusion output distributions and verifying it with an
  ancestor-only attention mask. It does not remove DFlash's trained
  block-diffusion drafter requirement.
- DFlare, arXiv:2606.02091 v2, extends DFlash by replacing the single shared
  fused target representation with per-draft-layer learnable fusion over a
  wider target-layer set. It reports 5.52x on Qwen3-4B, 5.46x on Qwen3-8B,
  and 3.91x on GPT-OSS-20B, improving DFlash by about 11%, 8%, and 5%.
- AngelSlim/Qwen3-8b-dflare model card: official DFlare draft checkpoint for
  Qwen/Qwen3-8B, 7 draft layers, hidden size 4096, 32 attention heads, 8 GQA
  KV heads, block size 16, target fusion layers [1, 5, 9, 13, 17, 21, 25, 29,
  33], BF16, and a benchmark script that reports speedup, acceptance length,
  and acceptance histograms.

## Molt Gap Map

- `src/molt/gpu/dflash/contracts.py` is a good contract shell: it names
  required conditioning fields and separates `draft_step` from `verify_step`.
  Gap: those fields are not consumed by a paper-faithful drafter.
- `src/molt/gpu/dflash/adapters.py` is a fail-closed adapter registry. Gap:
  adapter specs do not validate trained checkpoint identity, target model id,
  provenance, target hidden layers, block size, or attention capability.
- `src/molt/gpu/dflash/runtime.py` performs generic greedy block speculation
  and conditioned orchestration. Gap: no block diffusion, target hidden-state
  fusion, per-layer KV injection, anchor-conditioned mask, or trained drafter
  loading.
- `src/molt/gpu/generate.py` correctly avoids exporting generic speculation as
  DFlash and fails closed when no adapter exists for an explicit GPU backend.
- `src/molt/stdlib/tinygrad/dflash.py` correctly raises `ImportError` instead
  of aliasing generic tinygrad speculative helpers into the DFlash namespace.
- `src/molt/gpu/kv_cache.py`, `src/molt/gpu/transformer.py`, and
  `runtime/molt-runtime/src/builtins/gpu.rs` have attention/mask substrate but
  no DFlash-specific non-causal block attention or KV-injection proof.
- Tests currently cover scaffolding/fail-closed behavior in
  `tests/test_gpu_generate.py`. There is no dedicated
  `tests/gpu/dflash/test_dflash_fidelity.py` gate for target conditioning,
  per-layer KV injection, block diffusion, losslessness, and trained drafter
  provenance.
- `docs/spec/areas/perf/0520_DFLASH_CONTRACT.md` and
  `docs/design/foundation/67_compat_tinygrad_dflash.md` are mostly aligned.
  `docs/architecture/gpu-primitive-stack.md` and
  `docs/design/foundation/16_cpython-surface-stdlib-gpu-gap-audit.md` contain
  stale wording that can overclaim DFlash completeness or conflate DFlash with
  fused attention/DDTree.

## Next Structural Moves

1. Add `src/molt/gpu/dflash/SPEC.md` with machine-checkable F1-F6 fidelity
   requirements: target conditioning, per-layer KV injection, drafter/verifier
   separation, block diffusion, losslessness, and trained drafter provenance.
2. Strengthen `DFlashAdapterSpec` with target model id, draft model id,
   checkpoint digest, trained provenance, block size, target hidden-layer ids,
   attention backend capabilities, and DFlash/DFlare architecture family.
3. Add deterministic reference tests in `tests/gpu/dflash/test_dflash_fidelity.py`
   before claiming any real DFlash implementation. Include Dense and TurboQuant
   non-causal block-mask parity.
4. Implement the tiny reference drafter only after the spec/tests exist:
   target hidden-feature fusion, per-layer KV injection, anchor/last-token
   conditioning, non-causal masked block attention, and block logits.
5. Fix stale DFlash docs in the same arc so no roadmap/status text claims
   generic speculation as DFlash.
