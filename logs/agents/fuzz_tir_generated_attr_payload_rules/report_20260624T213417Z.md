# fuzz_tir_generated_attr_payload_rules

## Scope

- Extended `fuzz_tir_opcode_shapes` so the TIR pass fuzzer's registry-owned shape includes `attr_payload` rules for synthetic constant attributes.
- Generated typed Rust facts:
  - `FuzzTirAttrPayloadRule`
  - `FuzzTirOpcodeShape`
  - `opcode_fuzz_tir_attr_payload_rule_table`
- Migrated `runtime/molt-backend/fuzz/fuzz_targets/fuzz_tir_passes.rs` away from a local `match opcode` over `ConstInt` / `ConstFloat` / `ConstBool`.
- Switched generated `ConstFloat` fuzz attrs to canonical `f_value`, matching SCCP's float seed rule while existing lowerers still accept `value` as legacy input.
- Updated generator tests and docs for the new payload-rule axis.
