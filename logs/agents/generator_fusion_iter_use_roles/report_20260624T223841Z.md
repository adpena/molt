# Generator Fusion Iterator-Use Roles

## Summary

Moved the `generator_fusion.rs` raw-iterator use scanner off a private
`IterNext` / `Is` opcode match and onto the generated op-kind registry.

## Structural Changes

- Added `generator_fusion_iter_use_roles` to
  `runtime/molt-tir/src/tir/op_kinds.toml`.
- Added a dedicated generator validator that rejects missing rows,
  unknown fields, duplicate/unknown opcodes, `none` roles, and role/opcode
  drift.
- Generated `GeneratorFusionIterUseRole` and
  `opcode_generator_fusion_iter_use_role_table`.
- Updated `generator_fusion.rs` to consume the generated role table while
  preserving live operand-position and terminator-use proof in the pass.
- Extended `tests/test_gen_op_kinds.py` to prove TOML rows, generated Rust arms,
  production pass delegation, and validator drift rejection.
- Updated op-kind registry/spec docs and regenerated the structural audit board.
