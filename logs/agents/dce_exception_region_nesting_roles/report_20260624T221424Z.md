# DCE Exception Region Nesting Roles

## Summary

Moved DCE try-region depth tracking off a pass-local `TryStart` / `TryEnd`
opcode match and onto the generated op-kind registry.

## Structural Changes

- Added `exception_region_nesting_roles` to
  `runtime/molt-tir/src/tir/op_kinds.toml`.
- Added a dedicated generator validator for the role table. The validator
  rejects missing rows, unknown fields, duplicate/unknown opcodes, `none` roles,
  and role/opcode drift such as assigning the `exit` role to `TryStart`.
- Generated `ExceptionRegionNestingRole` and
  `opcode_exception_region_nesting_role_table`.
- Updated `dce.rs` to consume the generated table for try-depth tracking while
  keeping DCE responsible for traversal and dead-op policy.
- Extended `tests/test_gen_op_kinds.py` to prove TOML shape, generated Rust
  arms, validator failure behavior, and absence of production DCE
  `OpCode::TryStart` / `OpCode::TryEnd` match arms.
- Updated op-kind registry/spec docs and regenerated the structural audit board.
